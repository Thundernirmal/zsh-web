Tips Page Redesign Plan
Shadcn-Inspired Direction

Goal
- Redesign `/tips` so it feels like a polished, app-like knowledge surface instead of a long static archive.
- Improve scanability, filtering, discoverability, and visual hierarchy while keeping the page lightweight and accessible.

Recommended Direction
- Use a shadcn-inspired layout, not a full shadcn migration.
- Keep the current Astro + React + custom CSS stack for phase 1.
- Borrow the interaction patterns and composition ideas from shadcn:
  - Sidebar / inset layout for navigation
  - Command-style search and grouped results
  - Compact card composition with clear header/body/footer structure
  - Collapsible sections for categories

Why This Direction
- The current project does not use Tailwind or shadcn/ui directly, so a literal shadcn implementation would require a broader stack migration.
- A shadcn-inspired redesign gives the same clarity and feel with much lower risk.
- This lets the page improve quickly without forcing a design-system rewrite.

Design References Reviewed
- shadcn sidebar docs: collapsible sidebar, sticky header/footer, scrollable content, inset layout
- shadcn command docs: grouped search results, empty states, command-style filtering
- shadcn card docs: strong card composition with title, description, action, content, footer

Current Problems
- The page is a long list with 60 visible tips and little hierarchy.
- Every tip has almost the same visual weight, so the best tips do not stand out.
- Category anchor navigation works like a document, not like a modern app surface.
- Tips are plain text only, which makes them harder to scan and harder to connect to real commands.
- The page lacks a strong “entry point” such as featured tips, quick wins, or search-first discovery.

Proposed UX

1. Page Shell
- Add a clean page header with:
  - Title
  - Short value statement
  - Search input
  - Small meta summary like total tips and categories

2. Desktop Layout
- Use a two-pane app layout:
  - Left: sticky category rail with counts
  - Right: main content inset with search results, featured tips, and category sections

3. Mobile Layout
- Collapse the category rail into a horizontal filter row or a sheet-style category picker.
- Keep search at the top and preserve deep links.

4. Hero / Featured Area
- Add a “Featured Tips” block at the top.
- Surface:
  - Quick Wins
  - Most Useful for Newcomers
  - Hidden Gems

5. Search Experience
- Replace the basic archive feel with a command-style filter experience.
- Search should support:
  - tip text
  - category
  - command names mentioned in the tip
  - tags
- Empty state should feel intentional, not just “no results”.

6. Category Navigation
- Replace plain anchor chips with a richer category list:
  - category name
  - count
  - active state
  - optional icon
- Sections should support deep linking and preserve `scroll-margin-top`.

7. Tip Card Design
- Convert each tip into a compact card with this structure:
  - Title
  - Short description
  - Example command or shortcut
  - Category badge
  - Optional tags
  - Optional “Related command” link back to `/commands`
- Visual style should feel closer to shadcn cards:
  - tighter spacing
  - muted borders
  - clearer typography
  - subtle separators

8. Progressive Disclosure
- Do not show all 60 tips with equal prominence on first load.
- Show top 3 to 5 items per section initially.
- Add “Show More” inside each category.
- For long result sets, use `content-visibility: auto` and avoid heavy rendering.

Recommended Information Model
- The current `tips.json` shape is too minimal for a richer UI.
- Move toward optional structured fields:
  - `id`
  - `title`
  - `text`
  - `category`
  - `command`
  - `tags`
  - `level` (`beginner`, `intermediate`, `advanced`)
  - `featured`
  - `relatedCommands`

Migration Strategy
- Phase 1: keep compatibility with existing `{ text, category }` entries
- Phase 2: enrich the highest-value tips first with titles, commands, and tags
- Phase 3: gradually expand the rest of the dataset

Suggested Page Structure
- Header
- Search / filter toolbar
- Featured tips card group
- Main layout
  - left category rail
  - right content pane
- Category sections
  - section header
  - compact tip cards
  - show more control

Visual Style Notes
- Use calmer, flatter surfaces closer to shadcn:
  - softer border contrast
  - less decorative color noise
  - stronger use of spacing and typography instead of color
- Reduce emoji dependence in headings and labels.
- Prefer small badges and muted descriptions over bright accent-heavy chips.
- Use balanced heading wrapping and cleaner card rhythm.

Accessibility / Quality Requirements
- Use `:focus-visible` for custom focus treatment
- Add `aria-live="polite"` where search result counts update
- Preserve keyboard-friendly navigation
- Honor `prefers-reduced-motion`
- Add `scroll-margin-top` for deep-linked category sections
- Keep filters URL-synced so state is shareable

Implementation Plan

Phase 1: Content + structure
- Define the richer tip data model
- Identify featured tips and quick-win tips
- Normalize categories and add slugs / ids

Phase 2: Layout shell
- Replace the current single-column archive with a sidebar + inset layout
- Build responsive category navigation
- Add the new page header and toolbar

Phase 3: Search + filtering
- Upgrade search into grouped, command-like filtering
- Sync search and category state to the URL
- Add a proper empty state

Phase 4: Tip card system
- Build a reusable tip card component
- Add optional command snippets and related command links
- Add progressive reveal per category

Phase 5: Polish
- Refine spacing, borders, hover states, and focus states
- Improve performance for long lists
- Tighten mobile behavior and reduced-motion handling

Recommended Scope for First Implementation
- Do not install full shadcn/ui yet
- Do not migrate the whole site
- Redesign only `/tips`
- Reuse current theme tokens and create shadcn-like composition with local CSS

Nice-to-Have After Redesign
- Copy button for command snippets
- “Open in Commands” links for related aliases/functions
- Keyboard shortcut to focus search
- Featured tip rotation or “Tip of the Day”

My Recommendation
- Build a shadcn-inspired “Tips Playbook” with a sticky category rail, command-style search, featured tips, and compact tip cards.
- This is the best balance of visual upgrade, usability, and implementation risk for the current codebase.
